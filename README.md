학부생 시절 풍부한 개발팀 리더 역할을 맡았던 순간들은 "팀의 문제는 곧 나의 문제"라는 큰 부담을 안겨주었지만,    
그와 동시에 문제 해결 역량을 한단계씩 올려주는 귀중한 경험들 이었습니다.

<주요 문제 해결 이력> 
1. 중요 시간대 AI 서버 GPU 폭등 문제로 잦은 장애 이슈    
-> AI 서버 장애 자동 대응 시스템 구축 : 2주간의 알파 테스트 기간 중 AI 기능 장애 보고 0건 달성
2. LLM API 할루시네이션으로 인한 서비스 신뢰도 하락 이슈    
-> 할루시네이션이 발생하지 않는 역할로 LLM을 제한 : 2주간의 알파 테스트 기간 중 이슈 보고 0건
3. 메인 기능을 개발하던 중도 하차 팀원의 Service 클래스 품질 이슈    
-> Service 클래스는 흐름 제어만 담당, Manager/Helper 클래스로 역할 분담 : Service 클래스의 책임, 길이 80% 경량화
4. 외부 API의 호출 가능한 데이터 수 한계로 인한 데이터 유실 이슈    
-> 실시간 발생 데이터 모니터링 및 자동 Batch 시스템 구축 : 1주간 베타 테스트 기간 중 데이터 유실률 0% 달성
5. Valkey 기반 세션 서버 장애 예방 필요성     
-> Primary-Replica 구조 + Valkey Sentinel을 통한 세션 서버 장애 자동 대응 시스템 구축 : 알파 테스트를 통한 고가용성 검증
6. 세션/캐시 서버 분할 필요성      
-> 초기 설정, 운영 등에 차이를 부여하여 분할 아키텍처 구축 : 장애 격리, 서버 최적화 가능
7. 불변, 자주 조회되는 데이터에 대한 캐싱 처리 필요성     
-> Redis, Valkey 기반 캐싱 전용 서버 구축 : 캐싱을 통한 API 평균 응답 속도 95% 이상 개선 경험 3회 
8. 자체 제작 AI 모델에 대한 효율적인 관리 필요성     
-> Fast-API 구축 및 WebClient를 통한 Spring 서버 연동 : 장애 격리, 배포 독립성 보장

6. **Sharding을 통한 캐시 성능이 향상된 고가용성이 보장된 Valkey Cache Server**
